form 태그 역할 및 동작

form 역할

  1. 웹 사용자의 의견이나 정보를 받을 때 사용하는 큰 틀.
  2. form으로 입력된 사용자 데이터 전체를 서버로 전송가능.
  3. 서버 프로그램은 form으로 들어온 데이터를 사용하여 html 페이지를 구성하고 웹 브라우저에 전송후 출력.

form 동작 순서

  1. 사용자가 form으로 구성된 페이지에 데이터를 입력 후 submit
  2. form에 입력된 데이터가 서버로 전송됨
  3. 서버에 있는 서버 프로그램이 form 데이터 처리 후 html 페이지 구성
  4. 구성된 html 페이지를 웹 브라우져로 보냄
  5. 웹 브라우져에서 출력하여 사용자에게 보여짐

form 태그 속성

  1. action : form을 전송할 서버 쪽 스크립트 파일을 지정함
  2. name : form을 식별하기 위한 이름 지정
  3. accept-charset : form 전송에 사용할 문자 인코딩 지정
  4. target : action에서 지정한 스크립트 파일을 현재 창이 아닌 다른 위치에서 열도록 지정
  5. method : form을 서버로 전송할 http 메소드를 정함(GET 또는 POST 메소드)

*참고 : GET 과 POST 모두 서버로 데이터를 전송하느 역할. 단 GET은 URL에 데이터 노출. POST는 숨겨짐.

input 태그 속성

  1. type : text, radio, checkbox, password, button, hidden, fileupload, submit, reset 등 지정 가능
  2. name : 태그 이름 지정 (같은 이름으로 묶으면 radio에서 하나만 선택됨)
  3. readonly : 태그 읽기전용으로 지정
  4. maxlength : 해당 태그 최대 글자 수 지정
  5. required : 해당 태그 필수 지정. submit에 반드시 필요한 정보
  6. autofocus : 웹 페이지 로딩시 지정된 해당 태그로 포커스 이동
  7. placeholder : 정보 입력 자리에 힌트 출력
  8. pattern : 정규표현식 (예. 이메일, 주민버호 등)으로 유효한 값만 입력받을 때
  9. value : input 태그의 value 속성은 input 요소의 초깃값(value)을 명시 (속성값에 따라 역할 다름)
    • button, reset, submit : 버튼 내의 텍스트를 정의함.
    • hidden, password, text : 입력 필드의 초깃값을 정의함.
    • checkbox, image, radio : 해당 입력 필드를 선택 시 서버로 제출되는 값을 정의함.
    또한, input 요소의 type 속성값이 “file”인 경우에는 value 속성을 사용할 수 없음
